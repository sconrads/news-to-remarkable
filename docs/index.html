What it does

A fully automated pipeline from RSS to reMarkable, running every morning before you wake up.

🔐

Full-text behind paywalls

Logs in to Schibsted (Aftenposten, VG, E24) and Morgenbladet with your credentials. Fetches complete article text, not just RSS summaries.

✂️

Quality filtering

Short news flashes are discarded. Only articles with substantial text (configurable word count) are included. The longest, most in-depth pieces are prioritised.

📰

Newspaper-style PDF

A clean, serif-typeset A4 PDF with a front page, table of contents with clickable links, daily quote, and breaking news headlines.

📲

Delivered to reMarkable

Uploaded automatically to the /Nyheter folder on your reMarkable Cloud. Files older than a configurable number of days are cleaned up.

⏰

Runs while you sleep

A cron job on your Raspberry Pi (or any Linux machine) triggers at 06:00 every morning. No intervention needed.

⚙️

Fully configurable

Choose which newspapers are active, how many articles per category, minimum article length, your name, and retention period — all in one config.py.

📅

Today's calendar at a glance

Pulls today's events from one or more Google Calendars (via private iCal URLs) and shows them on the front page below the headlines. No Google account required on the device.

What you need

Four things are required before you start.

📖

reMarkable tablet

Any reMarkable (1 or 2) connected to reMarkable Cloud. The PDF is delivered wirelessly — no USB cable needed.

📰

Newspaper subscription(s)

  • Aftenposten (Schibsted)
  • Morgenbladet
  • VG (Schibsted) — optional
  • E24 (Schibsted) — optional
🖥️

A machine to run it on

A Raspberry Pi (ARM64, Debian 11+) works great. Any Linux machine with Python 3.9+ and a stable internet connection will do.

🐍

Python 3.9+

Standard Python installation. All dependencies are installed via pip from requirements.txt.

Setup guide

From zero to a fresh newspaper on your reMarkable every morning.

1

Clone the repository

Clone to your machine (Raspberry Pi or Linux desktop):

git clone https://github.com/sconrads/news-to-remarkable.git
cd news-to-remarkable
2

Create a virtual environment and install dependencies

python3 -m venv venv
venv/bin/pip install --upgrade pip
venv/bin/pip install -r requirements.txt

If WeasyPrint fails, install the required system libraries first:

sudo apt install -y libpango-1.0-0 libpangoft2-1.0-0
3

Install the Playwright browser

A bundled Chromium instance is used for authenticated article fetching. Install it inside the project folder to avoid conflicts:

PLAYWRIGHT_BROWSERS_PATH=~/news-to-remarkable/browsers \
  venv/bin/playwright install chromium
4

Configure your credentials

Copy the example file and fill in your subscription credentials:

cp .env.example .env
nano .env
SCHIBSTED_EMAIL=your@email.no
SCHIBSTED_PASSWORD=yourpassword
MORGENBLADET_EMAIL=your@email.no
MORGENBLADET_PASSWORD=yourpassword
Your credentials are stored only in .env on your own machine — never committed to git or sent anywhere.
5

Add your Google Calendar (optional)

To show today's events on the front page, add one or more private iCal URLs to .env. In Google Calendar, go to Settings → [calendar name] → "Secret address in iCal format" and copy the URL.

CALENDAR_ICS_URLS=https://calendar.google.com/calendar/ical/abc123.../private-.../basic.ics

Multiple calendars can be added as a comma-separated list. Leave the variable empty or unset to disable the calendar section entirely.

The iCal URL is a private, unguessable link. It does not require authentication and is never shared with anyone.
6

Set your name in config.py

Open config.py and change OWNER_NAME to your name. The front page will read "[Your name]s nyhetsmorgen".

OWNER_NAME = "Stian"
7

Download rmapi and register with reMarkable Cloud

Download the rmapi binary for your architecture from github.com/ddvk/rmapi/releases and place it in the project folder:

# ARM64 (Raspberry Pi):
wget https://github.com/ddvk/rmapi/releases/download/v0.0.32/rmapi-arm64.tar.gz
tar -xzf rmapi-arm64.tar.gz && chmod +x rmapi && rm rmapi-arm64.tar.gz

# x86-64 (regular Linux):
wget https://github.com/ddvk/rmapi/releases/download/v0.0.32/rmapi-amd64.tar.gz
tar -xzf rmapi-amd64.tar.gz && chmod +x rmapi && rm rmapi-amd64.tar.gz

Then register once with reMarkable Cloud. Get a one-time code at my.remarkable.com/connect/desktop, then run:

venv/bin/python register.py

Verify the connection:

./rmapi ls /
8

Run manually to verify

PLAYWRIGHT_BROWSERS_PATH=~/news-to-remarkable/browsers \
  venv/bin/python main.py

If everything is configured correctly, you should see the pipeline run and end with:

=== Ferdig! PDF sendt til reMarkable. ===

Check your reMarkable — the PDF should appear in the /Nyheter folder within seconds.

9

Schedule the daily cron job

Open crontab and add the line below to run every morning at 06:00:

crontab -e
0 6 * * * cd ~/news-to-remarkable && \
  PLAYWRIGHT_BROWSERS_PATH=~/news-to-remarkable/browsers \
  ~/news-to-remarkable/venv/bin/python main.py \
  >> ~/news-to-remarkable/cron.log 2>&1

Logs are written to ~/news-to-remarkable/cron.log.

Configuration reference

All settings live in config.py. No code changes needed.

Setting Default Description
OWNER_NAME "Stian" Your name. Appears in the front page title as "Names nyhetsmorgen".
ENABLED_SOURCES ["aftenposten", "morgenbladet"] Which newspapers to fetch. Available: aftenposten, morgenbladet, vg, e24.
MAX_ARTICLES_PER_CATEGORY 3 Maximum number of articles shown per category in the PDF.
MIN_ARTICLE_WORDS 300 Articles shorter than this word count are discarded as "news flashes".
MAX_BREAKING_HEADLINES 10 Number of latest headlines shown on the front page right column.
PDF_RETENTION_DAYS 5 PDFs older than this many days are automatically deleted from reMarkable.
REMARKABLE_FOLDER "/Nyheter" The folder on reMarkable Cloud where PDFs are uploaded.
CALENDAR_ICS_URLS unset Comma-separated list of private Google Calendar iCal URLs. When set, today's events appear on the front page below the headlines. Leave empty to disable.
